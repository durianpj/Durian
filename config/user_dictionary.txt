# OpenSearch /_analyze API (nori_tokenizer) 로 실제 토큰 분리 테스트 후 작성
# 쪼개지는 단어만 등록 (한 토큰으로 유지되는 단어는 등록 불필요)
# 테스트 방법: POST /_analyze { "tokenizer": "nori_tokenizer", "text": "단어" }
#
# 아래 단어들만 등록한 이유:
#   - 의미없는 조각으로 쪼개짐  예) 백엔드팀 → 백+엔+드팀, 우수사원상 → 우+수사+원상
#   - 부정 접두사가 분리됨      예) 미가입 → 미+가입, 비해당 → 비+해당
#   - 핵심 복합어가 분리됨      예) 사업장위치 → 사업+장+위치, 대학원졸 → 대학+원+졸
